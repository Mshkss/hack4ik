# Реальная карта и расчетная логика MVP

## Что было доработано

Приложение переведено с абстрактной SVG-схемы на интерактивную карту:

- подложка: нейтральная светлая CARTO Positron без картографических подписей через Leaflet;
- расчетный граф отображается поверх карты;
- пользователь может поставить старт и финиш кликом по карте;
- кликнутые точки привязываются к ближайшим узлам графа;
- выбранный C++ маршрутом путь подсвечивается;
- узлы маршрута отображаются как точки на карте;
- сложные поверхности показываются пунктиром;
- в подсказках ребер видны поверхность, длина, риск и коэффициент расхода.

## Какие сценарии кейса закрываются

| Сценарий | Как закрыт |
| --- | --- |
| Подготовка карты | есть геопривязанный слой `app_data/route_graph_georef.geojson` |
| Маршрут A-Б | старт/финиш выбираются в интерфейсе, расчет идет через C++ |
| Сравнение режимов | таблица сравнивает 4 режима для 2 конфигураций |
| Сложные участки | hard-поверхности запрещаются без поддува и подсвечиваются |
| Конфигурация лодки | `без поддува` и `с поддувом` меняют допустимость и расход |
| Карточка маршрута | показываются длина, время, топливо, остаток, резерв, риск |
| Объяснимость | показаны формулы, коэффициенты и предупреждения |
| Самостоятельная постановка точек | пользователь кликает на карте, точка привязывается к ближайшему узлу графа |

## Как карта превращается в граф

В MVP карта разбивается на расчетный граф автоматически:

1. `scripts/import_pipisa.js` читает объявленные типы поверхностей из JSON.
2. Если в JSON нет `map.zones`, импортёр создает стартовые демонстрационные полигоны зон: водоем/лед, открытая вода, камни, болото, шуга, мелководье, трава/камыш.
3. Эти зоны сохраняются в `app_data/surface_zones.geojson`.
4. Импортёр накладывает на карту сетку и классифицирует точки/ребра по полигонам зон.
5. Получается рабочий граф `auto_grid_from_surface_zones`.
6. C++ движок считает маршрут по этому автоматически построенному графу.
7. Ребра `map.edges` из пользовательского JSON теперь сохраняются только как тестовый reference, а не используются как рабочий граф.

Сейчас граф содержит:

- 299 узлов;
- 1058 ребер;
- 7 типов поверхностей;
- 1357 GeoJSON-объектов в `app_data/route_graph_georef.geojson`.

## Ручная постановка точек

Пользователь может:

- нажать `Поставить старт на карте`;
- кликнуть в нужную область;
- приложение найдет ближайший узел графа;
- покажет линию привязки от пользовательской точки до узла;
- пересчитает маршрут через C++.

То же работает для финиша. Это сделано так, потому что расчет выполняется по графу, а не по произвольной прямой на карте.

Также поддержан быстрый сценарий без прокрутки к кнопкам: первый клик по карте ставит старт, второй клик ставит финиш. После клика приложение показывает:

- координату пользовательской точки;
- ближайший узел графа;
- расстояние привязки до узла;
- пересчитанный маршрут.

Масштаб карты после пересчета не сбрасывается, чтобы пользователь мог сам удерживать нужный участок.

## Что использует расчет

C++ движок получает нормализованный файл:

```text
app_data/scenario.route
```

В нем лежат:

- профиль лодки;
- конфигурации;
- поверхности;
- режимы;
- ребра графа.

Для каждого ребра движок использует:

```text
time_segment = km / speed_kmh
fuel_segment = km * base_l_per_km * k_surf * k_load * k_mode
risk_segment = km * surface_risk
reserve_l = tank_l * reserve_frac_tank
```

## Что показывает UI

В интерфейсе отображаются:

- нейтральная светлая CARTO-подложка без картографических подписей;
- локальная векторная подложка коридора, чтобы карта оставалась читаемой при медленной загрузке тайлов;
- расчетный граф;
- активный маршрут;
- пользовательские точки старта/финиша и линии привязки к графу;
- карточка итогов;
- предупреждения;
- таблица участков;
- таблица сравнения режимов;
- блок "Что используется в расчете" с формулами и коэффициентами.
- легенда типов поверхностей;
- таблица всех поверхностей графа с количеством ребер и километражом.

## Ограничения

- Подложка карты зависит от CARTO/OSM-тайлов без подписей и требует интернет.
- Расчет выполняется локально и не зависит от сети.
- Промежуточные координаты синтетические.
- Данные не являются navigation-grade и не предназначены для реальной навигации.
