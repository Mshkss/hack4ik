# 02. Модель Raptor 650 для расчётов

## Зачем этот блок

В кейсе Raptor 650 выступает не как точная инженерная модель, а как набор параметров, влияющих на маршрут: запас топлива, расход, груз, проходимость и возможность пройти сложные участки с поддувом.

## Исходные параметры из постановки

| Параметр | Значение |
| --- | --- |
| Длина | 6,9 м |
| Ширина | 2,45 м |
| Вместимость | до 6 человек |
| Двигатель | Honda J35, 280 л.с. |
| Объём топливного бака | 370 л |
| Базовый расход | 0,7-1 л/км |
| Грузоподъёмность | 1-1,5 т |
| Конфигурация | базовая / с поддувом |

Внешних открытых источников по именно такой модели Raptor 650 найти не удалось, поэтому эти параметры лучше считать данными соревнования и явно указать это в README.

## Как использовать параметры в MVP

Вводить не сложную физическую модель, а профиль лодки:

```json
{
  "name": "Raptor 650",
  "fuel_tank_l": 370,
  "base_fuel_l_per_km": 0.85,
  "capacity_people": 6,
  "payload_kg_range": [1000, 1500],
  "engine_hp": 280,
  "variants": ["base", "blower"]
}
```

Базовый расход лучше задавать диапазоном `0.7..1.0 л/км`, а в расчётах использовать среднее `0.85 л/км`, показывая пользователю, что это модельная оценка.

## Конфигурации лодки

| Конфигурация | Роль в расчёте | Эффект в интерфейсе |
| --- | --- | --- |
| Без поддува | базовая проходимость | некоторые сложные участки недоступны или рискованны |
| С поддувом | повышенная проходимость | часть маршрутов становится допустимой, но расход/риск могут измениться |

Для демонстрации полезен сценарий, где один и тот же маршрут:

- недопустим для базовой конфигурации;
- допустим с поддувом;
- но имеет больший расход или повышенный риск.

## Коэффициенты для MVP

| Фактор | Пример значения | Комментарий |
| --- | --- | --- |
| Пустая/лёгкая загрузка | `1.0` | базовая модель |
| Средняя загрузка | `1.1` | умеренно повышает расход |
| Полная загрузка | `1.25` | повышает расход и риск потери глиссирования |
| Поддув на сложной поверхности | `0.8` к риску | снижает риск непроходимости |
| Поддув по расходу | `1.05..1.2` | можно заложить умеренный штраф |

## Что проверить экспертно

- Реалистичен ли диапазон расхода `0.7-1 л/км` для разных поверхностей.
- Насколько поддув увеличивает расход.
- Какие поверхности становятся допустимыми только с поддувом.
- Какая загрузка критична для глиссирования.
- Какой минимальный топливный резерв считать безопасным: в постановке предложен ориентир `15-25%`.

## Рекомендация для прототипа

Сделать редактируемый профиль лодки. Даже если в MVP будет только Raptor 650, экспертам будет видно, что расчётная логика не зашита в интерфейс и может быть перенесена на другой борт.
