# 07. Offline-first подход

## Зачем этот блок

По постановке расчётная часть должна работать без постоянного подключения к интернету. Это критично для реального применения: маршрут может строиться в местах со слабой связью, а в перспективе логика должна переноситься на бортовой терминал.

## Что должно работать без сети

| Функция | Offline-данные |
| --- | --- |
| Выбор карты | локальный список карт |
| Отображение карты | локальная подложка или схема |
| Построение маршрута | локальный граф/сетка |
| Расчёт топлива | локальные параметры лодки и коэффициенты |
| Расчёт риска | локальная разметка зон |
| Сравнение режимов | локальная функция стоимости |
| Предупреждения | локальные правила |

## Что можно обновлять при появлении связи

- карты;
- погодные данные;
- справочники поверхностей;
- параметры лодки;
- сохранённые маршруты;
- внешние ограничения;
- новые коэффициенты расчётной модели.

Обновление должно быть дополнительной возможностью, а не обязательным условием запуска.

## Формат хранения

Для MVP:

```text
data/
  maps/
    enisey_demo.geojson
  boats/
    raptor_650.json
  coefficients/
    surfaces.json
    modes.json
  routes/
    saved_routes.json
```

Для более реалистичной версии:

- GeoPackage для карты и геоданных;
- SQLite для локальных параметров;
- JSON для простых настроек и сценариев;
- локальный кэш ошибок и журнал расчётов.

GeoPackage особенно подходит под offline-сценарии, потому что это один SQLite-контейнер для геоданных, рассчитанный на мобильные устройства и ограниченную связь.

## Статусы данных в интерфейсе

Пользователь должен видеть:

- карта загружена локально;
- параметры Raptor 650 доступны;
- расчётные коэффициенты доступны;
- последнее обновление данных;
- связь отсутствует, но расчёт работает;
- какие данные не удалось обновить.

## Риски

| Риск | Как снизить |
| --- | --- |
| Устаревшая карта | показывать дату данных |
| Неполная разметка | подсвечивать неизвестные зоны |
| Ошибка коэффициентов | держать таблицу коэффициентов в README |
| Нет GPS/датчиков | иметь ручной режим ввода |
| Сбой обновления | не ломать локальный набор данных |
