# 08. Перспектива бортового терминала

## Зачем этот блок

Кейс просит показать, как прототип может быть перенесён в бортовой вычислительный контур. Это не значит, что нужно реализовать интеграцию с реальными датчиками, но архитектура должна быть готова к такому переносу.

## Потенциальные источники данных на борту

| Источник | Что может давать |
| --- | --- |
| GPS/ГНСС | координаты, скорость, курс |
| Датчик топлива | текущий остаток топлива |
| Двигатель | обороты, температура, ошибки |
| Эхолот | глубина |
| IMU | ускорения, крен, вибрации |
| NMEA 0183 | текстовые сообщения навигационных приборов |
| NMEA 2000 / CAN | сеть морских датчиков и дисплеев |
| Signal K | web-friendly JSON-модель морских данных |

NMEA 2000 используется для соединения морских датчиков и дисплеев, работает поверх CAN и позволяет устройствам обмениваться данными в сети. NMEA 0183 старше и использует текстовые сообщения. Signal K полезен как открытый слой, который может привести разные морские источники к JSON-модели.

## Архитектурная идея

Разделить систему на модули:

```text
UI
  |
Application layer
  |
Routing engine
Fuel/risk calculator
Map data access
Boat profile
Sensor adapters
Local storage
```

Критично: расчёт маршрута, топлива и риска не должен зависеть от конкретного веб-интерфейса. Тогда UI можно заменить на экран бортового терминала, а данные - на реальные датчики.

## Минимальные адаптеры будущего

| Адаптер | MVP-имитация | Реальная версия |
| --- | --- | --- |
| Позиция | ручной старт/финиш | GPS/ГНСС |
| Топливо | ручной ввод | датчик топлива/NMEA |
| Параметры двигателя | мок-данные | CAN/NMEA 2000 |
| Глубина | слой карты | эхолот |
| Погода | локальный JSON | внешний сервис при связи |

## Что показать в презентации

- Схему модулей.
- Где сейчас используются мок-данные.
- Какие интерфейсы можно заменить на датчики.
- Что расчётная логика работает локально.
- Что карта, параметры лодки и коэффициенты лежат отдельно от UI.

## Риски

- Нельзя обещать автопилот: кейс про поддержку принятия решения.
- Реальные NMEA 2000 данные могут быть закрытыми или требовать адаптеров.
- Датчики могут отсутствовать, поэтому ручной режим должен оставаться.
- Бортовой экран требует крупного, простого UI.
