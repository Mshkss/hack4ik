# 06. UX цифрового штурмана

## Зачем этот блок

Интерфейс в кейсе вторичен по отношению к логике, но эксперт должен быстро понять, как карта, параметры лодки и режим маршрута влияют на итоговый путь. Поэтому UX должен быть не красивой панелью ради панели, а инструментом принятия решения.

## Основная задача пользователя

Пользователь должен за один сценарий:

1. Выбрать карту.
2. Указать старт и финиш.
3. Выбрать конфигурацию Raptor 650.
4. Указать загрузку и запас топлива.
5. Выбрать режим маршрута.
6. Построить маршрут.
7. Понять, можно ли ехать и почему система предлагает именно такой путь.

## Первый экран

Минимальный состав:

| Блок | Содержание |
| --- | --- |
| Карта | поверхность, маршрут, рискованные участки |
| Панель маршрута | старт, финиш, режим |
| Параметры лодки | базовая/поддув, загрузка, топливо |
| Итоги | длина, время, расход, остаток топлива, запас хода |
| Предупреждения | риск, топливо, потеря глиссирования |

## Карточка маршрута

Карточка должна отвечать на вопросы:

- Сколько ехать?
- Сколько топлива уйдёт?
- Останется ли резерв?
- Где рискованные участки?
- Где может потеряться глиссирование?
- Почему выбран этот маршрут?

Пример структуры:

```text
Маршрут: Быстрый
Длина: 12.4 км
Время: 28 мин
Расход: 18 л
Остаток: 352 л
Риск: средний

Почему так:
- путь короче на 2.1 км;
- есть 0.8 км шуги, расход повышен;
- глиссирование теряется на участке S4;
- топливный резерв соблюдён.
```

## Предупреждения

Предупреждения должны быть крупными и понятными:

| Тип | Текст |
| --- | --- |
| Топливо | Недостаточный резерв топлива |
| Риск | Участок с высоким риском |
| Глиссирование | Глиссирование может быть потеряно |
| Проходимость | Участок недоступен без поддува |
| Данные | Карта устарела или не загружена локально |

## UX-принципы

Для интерфейса штурмана особенно важны:

- пригодность к задаче: быстрый выбор маршрута, а не длинная настройка;
- самопояснение: пользователь понимает расчёт без чтения кода;
- контроль: можно менять режим, загрузку, конфигурацию и видеть эффект;
- устойчивость к ошибкам: система не строит опасный маршрут молча;
- читаемость: крупные значения, минимум мелких таблиц.

## Проверка UX

Мини-тест на 5 минут:

1. Дать человеку сценарий: "Построй безопасный маршрут из A в Б с полной загрузкой".
2. Засечь, понял ли он, где задать параметры.
3. Спросить, почему система выбрала маршрут.
4. Проверить, заметил ли он предупреждения.
5. Попросить сравнить безопасный и экономичный режим.
