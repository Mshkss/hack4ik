# Airboat Navigator MVP

Веб-приложение для демонстрации цифрового штурмана аэролодки. Просчет маршрута выполняется отдельным C++ движком, веб-часть только показывает данные и вызывает расчет.

## Запуск

```bash
npm run build
npm start
```

После запуска открыть:

```text
http://localhost:5173
```

## Что внутри

| Путь | Назначение |
| --- | --- |
| `cpp/route_engine.cpp` | C++ расчетный движок |
| `scripts/import_pipisa.js` | импорт пользовательского JSON в формат движка |
| `server.js` | локальный web/API сервер |
| `web/` | интерфейс |
| `app_data/` | нормализованные данные приложения |
| `data/` | исследовательские и seed-данные MVP |
| `system/real_map_mvp_implementation.md` | описание реализации реальной карты и расчетных артефактов |

## Модель расчета

Движок строит маршрут по графу из JSON:

- исключает сложные поверхности без поддува;
- считает время по длине и скорости поверхности;
- считает топливо по формуле из JSON;
- считает риск по поверхности и длине;
- выбирает стоимость по режиму: быстрый, экономичный, кратчайший, безопасный.

Данные синтетические и не предназначены для реальной навигации.

## Реальная карта

Интерфейс показывает маршрут поверх OpenStreetMap через Leaflet. Расчетный граф из пользовательского JSON геопривязан:

- `Дивногорск` - координата из OpenStreetMap Nominatim;
- `Бирюса` - привязка к объекту `Верхняя Бирюса` из OpenStreetMap Nominatim как ближайшей точке финишного района;
- промежуточные узлы - ручная геопривязка синтетического графа, чтобы можно было демонстрировать расчет на карте.

Файл геопривязанного графа:

```text
app_data/route_graph_georef.geojson
```

Подложка карты требует интернет для загрузки OSM-тайлов, но расчет маршрута выполняется локально через C++ бинарник.

## Постановка точек на карте

В интерфейсе есть кнопки:

- `Поставить старт на карте`;
- `Поставить финиш на карте`.

После клика по карте приложение:

1. ставит пользовательскую точку;
2. находит ближайший узел расчетного графа;
3. показывает линию привязки;
4. пересчитывает маршрут через C++ движок.

Все возможные пути графа постоянно видны на карте и окрашены по типам поверхностей: вода, лед, мелководье, трава/камыш, шуга/наледь, камни, болото.

## Автоматический граф

Рабочий маршрутный граф больше не берется из тестовых ребер JSON. Импортёр делает так:

1. строит или читает зоны поверхностей;
2. сохраняет их в `app_data/surface_zones.geojson`;
3. разбивает карту на сетку;
4. классифицирует узлы и ребра по зонам;
5. генерирует `app_data/route_graph_georef.geojson`;
6. передает C++ движку уже автоматически построенный граф.

Текущая генерация создает 299 узлов и 1058 ребер. Ребра `map.edges` из пользовательского JSON сохранены только как тестовый reference.
